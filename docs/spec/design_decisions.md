> 본 문서는 Notion "CNN 가속기 설계 스펙 변천사 및 최종 결정사항" 페이지의 핵심 결정사항을 요약합니다.

## 모듈 설계 결정

### BRAM 베이스 모듈
- **모듈명**: `dual_port_ram` (초기 `bram_sdp`에서 변경)
- **포트명**: `wr` / `rd` (설계자 코딩 스타일 우선)
- **이유**: 설계자가 직접 작성한 코드 기준 통일

### 메모리 모듈 3종
| 모듈 | 역할 | 초기화 |
|------|------|--------|
| `weight_mem.sv` | ROM (학습된 가중치 고정값) | `$readmemh("weights.hex", ...)` |
| `feature_mem.sv` | 런타임 R/W Ping-Pong 버퍼 | 불필요 |
| `output_mem.sv` | HEAD 출력 R/W 버퍼 | 불필요 |

### 라인버퍼 구현 방식
- **최종 결정**: 레지스터 배열 기반, **col 단위 즉시 shift** 방식
- **라인버퍼 2개**로 충분 (행 단위 복사/Circular 방식은 3개 필요)
- **이유**: 실제 크기 계산 후 레지스터로 충분, 구현 단순화

### 모듈 분리
- `line_buffer` + `window_reg` 분리 유지 (합치지 않음)
- **이유**: 모듈 간 연결 이해도 향상, 학습 목적

## 하드웨어 블록 결정 (Ver 5.0)

### conv_engine 블록 크기
- **입력 병렬**: 4채널 (In block)
- **출력 병렬**: 6채널 (Out block)
- **DSP**: 4 × 6 × 9 = **216개** (220개 한도 내 최대)
- **LCM(4,6) = 12** → 내부 채널을 12의 배수로 설계 → In/Out 낭비 0

### 채널 설계 (Ver 5.0)
```
L0: Conv(3→12)   L1: Conv(12→24)   L2: Conv(24→48)
L3: Conv(48→96)  HEAD: Conv(96→18, 1×1)
```

## BRAM Ping-Pong 버퍼 전략

- 버퍼 A (현재 레이어 입력) + 버퍼 B (현재 레이어 출력) 번갈아 사용
- 레이어 연산 완료 후 이전 피처맵 덮어쓰기 재활용
- 최대 피처맵: L1 출력 128×128×16 = 256KB → 버퍼 2개 = 512KB → ~114개 BRAM

## ⚠️ 중요 설계 원칙

1. **BRAM 낭비 방지**: `DATA_WIDTH=8`, `DEPTH=2^n` 원칙 준수
2. **합성 후 반드시 확인**: Vivado Utilization Report → Block RAM Tile ≤ 136개
3. **HEAD에 MaxPool 없음**: 공간 해상도 32×32 유지
4. **마지막 Conv(L3)에도 MaxPool 없음**: 최종 출력 해상도 보존
